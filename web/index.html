This is the X4's firmware — the same Rust that runs on the device's ESP32-C3 — compiled to WebAssembly. The screen above is its real framebuffer: the same Literata bitmaps, the same line wrap, the same pagination, drawn one bit deep at 800 × 480.

The hardware around it is pretend. The SD card is three small built-in books, the Wi-Fi sync session is scripted, and the e-ink panel is a canvas that flashes the way the real GDEQ0426T82 does: a quick 421 ms update for a page turn, a full inverting clean when ghosting builds up. Your place on the shelf is kept in this browser.
